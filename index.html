Introduction

In the tradditional way of display mathematical proof, we present it in a linearly way and read from line to line. This sometime can make that proof harder to go through, especially when it is not well documented since the the relations between proof's arguement is not always linear and it's harder to keep track all the references when the proof is more complicated.

In this website we aim to make that mathematical proof become easier to visualize and increase the readablity so that even machines can understand wether the proof is correct or not. To do so we interpret it as a directed acyclic graph where arguements are represented as nodes and the dependencies between them are represented as edges.

Getting startted

You can create your own proof immediatedly by visiting this link., To have the best expiriences, I recommend you use the full screen feature by clicking 'F11'

Use ctrl + click to create a node in the cursor position and popup the editor for you to edit that node, if in that position there is already the node, the editor will popup to edit the node you clicked in instead.

There are fews editing options you can use, such as \bf for making bold text, \it for making italic text, \enumurate and \itemize for making ordered list and unordered list respectively.

For typing math equation we use the MathJax library and the markup language we use is latex. Latex is a common markup language to render mathematic, it has two mode which is inline mode and display mode that have the delimeters $ and $$ respectively, for the full documentation about latex that MathJax supported check out this.

To structure your proof into lemmas and arguements you can double click a node to see it's full detail and treat those node as an sublemma. You can reference other node including nodes that defined outside of that sublemma by using \ref command or right-click into the node you want to reference from.
